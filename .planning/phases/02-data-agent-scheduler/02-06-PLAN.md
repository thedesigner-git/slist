<objective>
Fix the DB schema / ORM mismatch for agent_run_results.company_id nullability.

Purpose: The migration defines company_id as BIGINT NOT NULL with ON DELETE CASCADE, but the ORM model correctly has nullable=True with ON DELETE SET NULL. When a company fails before upsert_company() in runner.py, company_id is None. The DB NOT NULL constraint will reject the INSERT, losing the error log row and violating AGENT-07.

Output: A new Supabase migration that ALTERs the column to be nullable and changes the FK action to SET NULL, aligning DB schema with the ORM model.
</objective>
