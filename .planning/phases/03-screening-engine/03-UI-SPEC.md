## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 400 (regular) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 700 (bold) | 1.1 |

**Usage mapping:**
- Body (14px/400): Criterion description text, drawer helper text, threshold input labels
- Label (12px/400): Preset section headers ("GROWTH CRITERIA", "VALUE CRITERIA"), score badge text, criterion threshold unit labels (e.g., "%", "x") — differentiated from body via `uppercase` + `tracking-widest`, not weight
- Heading (20px/600): Drawer title ("Screening Criteria"), shortlist section title
- Display (28px/700): Not used in Phase 3 (reserved for Phase 4 dashboard hero stats)

**Weight constraint:** 2 weights in Phase 3: 400 (regular) and 600 (semibold) — medium (500) and bold (700) deferred to Phase 4. Labels at 12px use weight 400 and rely on uppercase + letter-spacing for visual differentiation instead of weight.

## Color

| Role | Tailwind Class | Hex | Usage |
|------|---------------|-----|-------|
| Dominant (60%) | bg-zinc-900 | #18181b | Page background, drawer backdrop |
| Secondary (30%) | bg-zinc-800 | #27272a | Drawer panel surface, criterion rows, score cards |
| Accent (10%) | bg-violet-500 | #8b5cf6 | See accent reserved list below |
| Success semantic | text-emerald-400 | #34d399 | Growth label badge, criterion PASS indicator |
| Warning semantic | text-amber-400 | #fbbf24 | Value label badge |
| Destructive | bg-red-600 | #dc2626 | Watch removal confirmation (single use) |

**Accent (violet-500) reserved for:**
1. Active/enabled Toggle Switch thumb and track
2. "Apply" CTA button in the criteria drawer
3. Score percentage badge background when score >= threshold (shortlisted)
4. Watch bookmark icon fill when active

**Accent is NOT used on:** Disabled states, secondary buttons, hover states (use zinc-700 for hover), text links, borders, separators.

**Color decision notes:**
- Violet chosen to match Revolut's bold accent language; distinguishable from the success green (emerald) and warning amber used for Growth/Value labels
- Score badge below threshold: bg-zinc-700 / text-zinc-300 (neutral, not red — failure is not alarming, just informational)
- Criterion PASS: emerald-400 checkmark icon, no background change on row
- Criterion FAIL: zinc-500 text, zinc-600 icon — subdued, not alarming

## Component Inventory

Phase 3 introduces these UI surfaces:

### 1. Criteria Configuration Drawer

**Component:** Radix `Sheet` from shadcn (side="right", width=380px on desktop)

**Trigger:** "Screening Criteria" button in dashboard header area — icon (SlidersHorizontal from lucide) + label, outlined style, zinc-700 border

**Structure:**
```
SheetHeader
  Title: "Screening Criteria"          (Heading / 20px / semibold)
  Description: "Thresholds apply to trailing twelve months (TTM)."  (Body / 14px / zinc-400)

SheetContent (scrollable, gap-2xl between preset sections)
  [Section: Growth Criteria]
    Row: preset-level toggle
      Label: "GROWTH"                   (Label / 12px / 400 / zinc-400 / uppercase / tracking-widest)
      Switch (Radix) — enabled = violet accent
    [Criterion rows, only visible when preset enabled]
      CriterionRow:
        Left: criterion name (14px/400/white) + description (12px/400/zinc-400)
        Right: ThresholdInput (number input, 64px wide) + unit label (12px/400/zinc-400) + individual Switch
  [Section: Value Criteria]
    Same structure as Growth

SheetFooter
  Button "Apply Changes"   (accent violet, full-width, semibold 14px)
  Helper text below: "Shortlist recalculates in the background."  (12px / zinc-500)
```

**States:**
- Preset disabled: all criterion rows fade to opacity-40, individual controls non-interactive
- Threshold input invalid (e.g., negative P/E): red-600 border, inline error "Must be a positive number"
- Apply button loading: spinner replaces label, button disabled, violet-600 background
- Apply success: button returns to default, drawer stays open (user can keep tweaking)

### 2. Score Badge

**Displayed per company on the shortlist row/card.**

Structure: Pill badge, inline.
- Text: "86%" or "6/7" — Label size (12px/400/uppercase)
- Padding: 4px vertical / 12px horizontal
- Shortlisted (score >= threshold): bg-violet-500/20 (10% violet), text-violet-300, violet-500/30 border
- Not shortlisted: bg-zinc-700, text-zinc-400, no border

### 3. Label Badges (Growth / Value)

Two independent badges, shown side-by-side when both apply.

- Growth: bg-emerald-500/15, text-emerald-400, 12px/400/uppercase
- Value: bg-amber-500/15, text-amber-400, 12px/400/uppercase
- Shape: rounded-full, 4px vertical / 8px horizontal padding

### 4. Watch Bookmark Toggle

**Single icon button.** Bookmark icon (lucide `Bookmark`) on each company row.

- Inactive: Bookmark icon, zinc-500, outline style
- Active (Watch): Bookmark icon, violet-500, filled style (`BookmarkCheck`)
- Touch target: 44px x 44px minimum (transparent hit area around 20px icon)
- No confirmation dialog on adding Watch
- Removal confirmation: Only when removing Watch from a company already on the shortlist purely due to Watch override (score below threshold). Inline popover: "Remove Watch? This company will leave the shortlist." with "Remove" (red-600) and "Keep" (zinc secondary) buttons.

### 5. Recalculation In-Progress Indicator

When the background scoring job is running after "Apply Changes":

- Slim 2px progress bar at the top of the shortlist panel, animating left-to-right in violet
- Supplementary text below shortlist heading: "Updating shortlist..." (12px/zinc-400)
- Disappears when recalculation completes (polling interval: 2 seconds via SWR/useEffect)

## Interaction Contracts

| Interaction | Behavior |
|-------------|----------|
| Open criteria drawer | Sheet slides in from right, 300ms ease-out |
| Toggle preset switch | Criterion rows fade in/out, opacity transition 150ms |
| Toggle individual criterion | Row dims to opacity-40 when disabled, threshold input becomes read-only |
| Change threshold value | No immediate feedback — applies only on "Apply Changes" |
| Click "Apply Changes" | POST to criteria API, button shows spinner, drawer stays open, shortlist shows recalculation indicator |
| Click Watch (inactive) | Immediate optimistic update — bookmark fills violet, PATCH /api/watch, no confirmation |
| Click Watch (active, score >= threshold) | Immediate remove, PATCH /api/watch |
| Click Watch (active, score < threshold — company would leave shortlist) | Inline popover confirmation before removing |
| Shortlist recalculation completes | Progress bar disappears, shortlist re-renders with updated scores and labels |

## Accessibility Notes

- All Switch components must have `aria-label` describing the criterion (e.g., "Toggle Revenue Growth criterion")
- Score badge must have `title` or `aria-label` with full text ("86% — 6 of 7 criteria passed")
- Watch bookmark button must have `aria-label="Add to Watch"` / `aria-label="Remove Watch"` toggling with state
- Watch bookmark button must also expose a visible tooltip matching aria-label copy (see Copywriting Contract)
- Drawer must trap focus when open (Radix Sheet handles this natively)
- Threshold number inputs: `type="number"`, `min="0"`, `aria-label` with criterion name
- Color contrast: all text on zinc-800/900 backgrounds meets WCAG AA (zinc-400 on zinc-800 = 4.6:1)
