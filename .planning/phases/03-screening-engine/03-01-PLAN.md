def _upsert(db: Session, model, data: dict, conflict_cols: list[str]):
    stmt = pg_insert(model).values(**data)
    update_cols = {k: stmt.excluded[k] for k in data if k not in conflict_cols}
    stmt = stmt.on_conflict_do_update(index_elements=conflict_cols, set_=update_cols)
    db.execute(stmt)

Create `apps/backend/agent/scoring.py` with three public functions:

**1. `compute_ttm_metrics(db: Session, company_id: int) -> dict`**
- Query the last 4 rows from `growth_metrics` table for the given company_id, ordered by period DESC
- Query the last 4 rows from `ratios` table for the given company_id, ordered by period DESC
- Compute TTM averages for these fields:
  - From growth_metrics: revenue_growth_yoy, eps_growth_yoy, roe (note: roe is in ratios table actually — use ratios.roe), fcf_margin
  - From ratios: pe_ratio, pb_ratio, debt_to_equity
- For each field, average all non-None values. If all values are None, return None for that field.
- Note on ROE: ROE is stored in ratios table (not growth_metrics). Use ratios.roe for the growth_roe criterion evaluation.
- Return dict: `{"revenue_growth_yoy": float|None, "eps_growth_yoy": float|None, "roe": float|None, "fcf_margin": float|None, "pe_ratio": float|None, "pb_ratio": float|None, "debt_to_equity": float|None}`

**2. `score_company(ttm: dict, settings: UserCriteriaSettings) -> dict`**
- Pure function (no DB access). Takes TTM metrics dict and user settings.
- Build criteria list from settings. Only include criteria where:
  - The parent preset is enabled (settings.growth_enabled / settings.value_enabled)
  - The individual criterion is enabled (settings.growth_revenue_growth_yoy_enabled etc.)
- Growth criteria (direction: ">" means metric must be greater than threshold):
  - revenue_growth_yoy > settings.growth_revenue_growth_yoy
  - eps_growth_yoy > settings.growth_eps_growth_yoy
  - roe > settings.growth_roe
  - fcf_margin > settings.growth_fcf_margin
- Value criteria (direction: "<" means metric must be less than threshold, except fcf_margin which is ">"):
  - pe_ratio < settings.value_pe_ratio
  - pb_ratio < settings.value_pb_ratio
  - fcf_margin > settings.value_fcf_margin
  - debt_to_equity < settings.value_debt_to_equity
- For each active criterion: if ttm value is None, count as FAIL (per D-04). Otherwise compare against threshold with the correct direction.
- Count passed and total active criteria. Score = passed / total (or 0.0 if total is 0).
- Determine growth_passed: if growth preset enabled, count only growth criteria — growth_passed = (growth_passed_count / growth_total_count) >= settings.shortlist_threshold. If growth preset disabled, growth_passed = False.
- Same logic for value_passed with value criteria.
- is_shortlisted = score >= settings.shortlist_threshold (per D-02)
- Return dict: `{"score": float, "criteria_passed": int, "criteria_total": int, "growth_passed": bool, "value_passed": bool, "is_shortlisted": bool}`

**3. `score_all_companies(db: Session, user_id: str) -> None`**
- Load UserCriteriaSettings for user_id. If none exists, create one with defaults (per D-08).
- Query all company IDs from `companies` table.
- For each company: compute_ttm_metrics, then score_company, then upsert into ShortlistScore.
- Upsert uses the existing `_upsert` pattern from store.py — conflict on (user_id, company_id).
- Preserve `is_watch` from existing ShortlistScore row if one exists (do not overwrite Watch bookmarks).
- Set is_shortlisted = score_result["is_shortlisted"] OR existing is_watch.
- Commit after all companies processed.
