```python
router = APIRouter(prefix="/api/criteria", tags=["criteria"])
```

**Endpoint 1: GET /api/criteria/settings**
- Depends: `get_current_user`, `get_db`
- Extract user_id from JWT payload: `user_id = user["sub"]`
- Query UserCriteriaSettings where user_id matches
- If none exists, create a new row with all defaults (per D-08 — seeded on first access) and commit
- Return the settings as a dict with all threshold and toggle fields
- Response shape: `{"growth_enabled": bool, "value_enabled": bool, "growth_revenue_growth_yoy": float, ..., "shortlist_threshold": float}`

**Endpoint 2: PUT /api/criteria/settings**
- Depends: `get_current_user`, `get_db`, `BackgroundTasks`
- Accept Pydantic model `CriteriaSettingsUpdate` with all optional fields (so users can update partial):
  - growth_enabled: bool | None = None
  - value_enabled: bool | None = None
  - All 8 threshold fields: float | None = None
  - All 8 enabled toggle fields: bool | None = None
  - shortlist_threshold: float | None = None
- Load existing settings (or create if not exists)
- Update only provided (non-None) fields using `setattr(settings, field, value)` loop over `model.model_dump(exclude_unset=True)`
- Commit settings immediately (per D-06 — settings save instantly)
- Trigger background recalculation: `background_tasks.add_task(score_all_companies_wrapper, user_id)` where `score_all_companies_wrapper` opens a new DB session, calls `score_all_companies(db, user_id)`, then closes session
- Return updated settings dict
- Return header or field `X-Recalculating: true` so frontend knows scoring is in progress

**Endpoint 3: PATCH /api/criteria/watch/{company_id}**
- Depends: `get_current_user`, `get_db`
- Toggle Watch on ShortlistScore for (user_id, company_id)
- If no ShortlistScore row exists, create one with score=0, criteria_passed=0, criteria_total=0, is_watch=True, is_shortlisted=True
- If row exists, flip is_watch. If is_watch becomes True, set is_shortlisted=True. If is_watch becomes False, set is_shortlisted based on score >= threshold (load user settings to check).
- Commit and return `{"company_id": int, "is_watch": bool, "is_shortlisted": bool}`

**Endpoint 4: GET /api/criteria/shortlist**
- Depends: `get_current_user`, `get_db`
- Query ShortlistScore WHERE user_id matches AND is_shortlisted = True
- Join with Company table to include ticker, name, market, sector
- Return list of: `{"company_id": int, "ticker": str, "name": str, "market": str, "sector": str|null, "score": float, "criteria_passed": int, "criteria_total": int, "growth_passed": bool, "value_passed": bool, "is_watch": bool}`

**Endpoint 5: GET /api/criteria/scores**
- Depends: `get_current_user`, `get_db`
- Query ALL ShortlistScore rows for user_id (shortlisted or not)
- Join with Company. Return full list with same shape as shortlist endpoint.
- Purpose: Frontend can show all companies and their scores, not just shortlisted ones.

**Endpoint 6: GET /api/criteria/status**
- Depends: `get_current_user`
- Return `{"recalculating": bool}` — check if a background task is running for this user
- Implementation: use a module-level dict `_recalc_in_progress: dict[str, bool] = {}`. The wrapper function sets `_recalc_in_progress[user_id] = True` before calling score_all_companies and `False` after.

Define the `CriteriaSettingsUpdate` Pydantic model at the top of the file (or in a separate schemas file if preferred, but keeping it in the router file is simpler for now).

Define `score_all_companies_wrapper(user_id: str)` as a module-level function that:
1. Sets `_recalc_in_progress[user_id] = True`
2. Opens `db = Session()`
3. Calls `score_all_companies(db, user_id)`
4. Closes db
5. Sets `_recalc_in_progress[user_id] = False`
